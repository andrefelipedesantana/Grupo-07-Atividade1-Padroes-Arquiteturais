// 2° Etapa: Análise manual da estrutura organizacional do repositório
// Durante a análise, foi identificado que o projeto segue um sistema baseado
// em microserviços, composto pelos seguintes módulos:

// • HuggingGPT — MONOREPO
//   Contém um front-end em Vue.js e um back-end em Python.
//   Responsável pela comunicação com modelos de IA e pela interface de interação.

// • TaskBench
//   Conjunto de arquivos em Python e JSON voltados à avaliação de desempenho
//   de LLMs em automação de tarefas, utilizando Tool Graphs para decomposição
//   e execução de fluxos de tarefas.

// • EasyTool
//   Módulo em Python que auxilia na geração e gerenciamento de ferramentas (tools)
//   utilizadas pelos outros serviços.
//   Atua como repositório e orquestrador de utilidades.

// ---------------------------------------------------------------

// 3° Etapa: Geração do arquivo de estrutura do projeto
